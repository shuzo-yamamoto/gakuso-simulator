# CLAUDE.md — 額装シミュレーター 絶対ルール

このファイルは実装AI（Claude Code ほか任意のコーディングエージェント）向けの指示書である。
以下は **モデルやセッションが変わっても常に遵守すべき本プロジェクトの絶対ルール**。

## プロジェクト概要
額縁・画材店（画箋堂）の店頭・工房で使う額装オーダーシミュレーター。コーナーサンプル画像と作品画像から額装完成イメージをCanvasで合成する。
- 構成: **単一HTMLファイル（`index.html`）のみ**。素のHTML/CSS/JS＋Canvas API。サーバーなし・完全クライアントサイド
- 対象端末: PC・iPad（Safari）
- 現行バージョン: v23（動作実績あり）。本プロジェクトは v24 への改修
- 注記: 本アプリはGASを使わない。共通標準 `00_GAS業務アプリ開発標準.md` のうちGAS固有条項（google.script.run・LockService・シートI/O等）は適用外。**設計書ファースト・段階実装・突合セルフレビュー・検収テストの思想は全面適用**する

## 大原則（最優先）
1. **仕様の唯一の正は `要件定義書_v24_画像前処理・エクスポート.md`**。コードと設計書が矛盾したら設計書が正
2. **確定事項は変更禁止**（設計書§8）。特に:
   - **生成AI（画像生成API等）は一切使用しない**（受注ツールの商品忠実性のため）
   - **後方互換**: 柄物モード・照明ムラ補正OFF・ミラータイリングOFF・テクスチャスケール100%で v23 と画素一致
   - **免責文言のエクスポート画像への焼き込みは必須・OFF不可**
   - エクスポート形式はPNG、ファイル名規則は設計書§3.5-4のとおり
3. **仕様外の機能追加は提案止まり**。ユーザーの承認なしにコード化しない
4. 仕様変更が必要なときは**先に設計書を更新してからコードを直す**
5. 順序を守る: 設計書を読む → 計画提示 → 承認 → ステップ実装 → セルフレビュー → 検収テスト

## 品質保証ルール
1. 設計書にない機能・項目・画面を勝手に追加しない。改善案は提案としてユーザーに伝え承認を得る
2. 各ステップ実装後、**設計書との突合セルフレビュー**を行い「設計書の該当箇所→実装箇所」の対応表を報告する
3. 処理A〜Eのアルゴリズムは設計書§3の数式・手順と一字一句一致させる（σの算出式・トリム平均の除外率・端数処理を含む）
4. **ライブラリ・フレームワーク・CDN読込を追加しない**（素のHTML/CSS/JS＋Canvas APIのみ）
5. コンテキストが長くなっても、各ステップ開始時に設計書の該当箇所を読み直す
6. 設計書の曖昧さ・矛盾を見つけたら、推測で埋めず必ず質問する
7. 各ステップの完了時に動作確認手順を README に追記する（引き継ぎ可能性の担保）

## 最重要制約
- **追加費用ゼロ**: 外部有料サービス・有料API・外部通信は一切使わない
- **単一HTMLファイル維持**: ファイル分割しない（現場ではファイル1つで配布するため）
- v23の既存機能（マット・面金・ライナー・入子・45°マイターカット描画）に一切のデグレを起こさない
- 初心者が保守するため、コードには日本語コメントを丁寧に入れる
- エラー・案内メッセージは日本語で、利用者が読んで対処できる文言にする

## 実装上の必須ルール
- 描画ロジックは `drawPreview()` から**スケール係数を引数に取る共通関数**に切り出す（プレビュー=1、エクスポート=2。設計書§3.5-1）
- 画像処理（処理A/B/C/D）は切り出し帯確定時（`buildTex()` 相当）に適用し、`tileTopBot` / `tileSide` の既存タイル機構に載せる
- 定量検証関数 `checkStripUniformity()` をグローバル公開し、ブラウザconsoleから実行できるようにする（設計書§7.1）
- 無地モード時は「テクスチャスケール」スライダーをグレーアウト
- iPad Safari対応: download属性が効かない場合のフォールバック（新規タブ＋長押し案内）を必ず実装

## Git運用（設計書§5.2）
- Step完了時のコミットは必須。設計書変更が伴う場合は**設計書コミット→実装コミットの順**
- コミットメッセージは日本語（例: `Step1: 無地モード（プロファイル押し出し）実装`）
- **コミット・pushは突合セルフレビュー報告→ユーザー承認後**に行う。push前にコミット内容の要約を報告する
- 改修着手前に作業ツリーがクリーン（コミット済み）であることを確認する
- タグ: 各Step実機確認完了時 `v24-step1` 等、全Step完了時 `v24.0`

## 検収テスト
- **設計書§7の検収テスト T-1〜T-10 を各Stepの完了条件に従いクリアする。**パスするまで次のStepに進まない
- 検収用画像は `test-images/` に格納済み:
  - `frame1.jpg` = 画像①（メーカー画像・柄物）
  - `IMG_3569.jpg` = 画像②（現場撮影・無地）
  - `result_v23_maker.png` / `result_v23_genba.png` = v23での結果（比較基準）
- 手動チェックリストは設計書§7.3

## 実装ステップと完了条件
- **Step 1** 無地モード（処理B: プロファイル押し出し）＋種類切替UI: 完了条件 = T-1・T-2クリア
- **Step 2** 照明ムラ補正（処理A）＋ミラータイリング（処理C）＋補正UI: 完了条件 = T-3〜T-6クリア
- **Step 3** 4点パース補正（処理D）: 完了条件 = T-7クリア
- **Step 4** エクスポート機能（処理E）＋保存UI: 完了条件 = T-8〜T-10クリア
  - ※Step 4は独立しているため、ユーザーの指示があればStep 1の直後に前倒し可

各ステップの完了条件3点セット: ①コード生成 ②突合セルフレビュー報告 ③検収テスト該当項目クリア。
揃うまで次のステップへ進まないこと。
