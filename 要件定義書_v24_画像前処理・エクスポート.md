# 額装シミュレーター 要件定義書 追記案 — 画像前処理仕様

- 作成日: 2026-07-18
- 対象: 額装シミュレーター（現行 v23 → 本追記実装後 v24）
- 対象業務: 店頭・工房での額装オーダー受注時のシミュレーション
- 実装者への注記: 本書は特定のAIモデル・開発者に依存せず実装できるよう、処理ルールをすべて明文化し、§7に実データによる検収テストケースを収録している。**画像処理ロジックは本書を唯一の正とし、コード内に独自解釈やマジックナンバーを持ち込まないこと。**

---

## 1. 追記の目的

### 1.1 背景と課題
- メーカー提供のコーナーサンプル画像（均一照明・正対撮影）ではシミュレート品質に問題がない
- 実店舗・工房でスマートフォン撮影した画像（例: `test-images/IMG_3569.jpg`）では以下の理由で合成感が強く出る
  1. **照明勾配**: 辺の左右・上下で明るさが異なり、タイリング時に明暗の縞が周期的に現れる
  2. **無地フレームへの不要なタイリング**: 無地（単色・シンプル形状）のフレームでもテクスチャを繰り返し貼るため、撮影ノイズがそのまま繰り返される
  3. **パースずれ**: 正対撮影でないため切り出し矩形がフレーム辺と平行にならない

### 1.2 解決方針
- **実写画素の保存的変換のみで解決する**（照明の平坦化・統計処理・幾何変換）
- 生成AIによる画像合成・補完は使用しない（→ §8 確定事項一覧）

### 1.3 あわせて追加する機能
- **シミュレート結果の画像エクスポート**（§3.5 処理E）: 完成イメージをPNGで保存し、お客様への提示・受注控え・メール共有に使えるようにする

### 1.4 スコープ外
- **生成AI（Gemini・GPT Image 等）による画像生成・編集・補完**。理由: 出力が元画像の再創作であり、彫刻パターンの寸法・細部が実物と一致する保証が原理的にないため、受注ツールの商品忠実性要件と両立しない。マーケティング素材等、受注と切り離した用途で使う場合も本アプリには組み込まない
- 外部ライブラリ・CDN読込の追加（素のHTML/CSS/JS＋Canvas APIのみを維持）
- サーバーサイド処理（現行どおり単一HTMLファイル・完全クライアントサイドを維持）
- フレーム種類（無地/柄物）の自動判定（利用者が手動選択する。§4.1）

## 2. 用語定義

| 用語 | 定義 |
|---|---|
| 切り出し帯 | 利用者が2点クリックで指定した矩形テクスチャ（現行 `texRect`。幅 sw × 高さ sh） |
| 長手方向 | 切り出し帯の長い方の辺の方向。sw ≥ sh なら横（x方向）、sw < sh なら縦（y方向） |
| 断面方向 | 長手方向と直交する方向。フレームの断面プロファイルに相当する |
| 照明マップ | 切り出し帯から推定した低周波の明るさ分布 L(x,y) |
| プロファイル列 | 断面方向の各位置について長手方向に画素を統計集約した1画素幅の列 |

## 3. 処理仕様（正本）

### 3.1 処理A: 照明ムラ補正（フラットフィールド補正）

適用対象: 無地モード・柄物モードの両方（切り出し帯確定直後、他の全処理より先に実行）。

1. 切り出し帯の各画素の輝度を `Y = 0.299R + 0.587G + 0.114B` で算出し、グレースケール画像を作る
2. グレースケール画像に**ガウスぼかし**をかけ照明マップ L(x,y) を得る
   - 標準偏差 σ = `max(sw, sh) / 4`（下限 8px）
   - 実装手段（畳み込み／複数回ボックスぼかし近似）は問わないが、近似の場合はボックスぼかし3回反復以上とする

   **実装パラメータ（v24で確定・2026-07-20追記）**

   実データでは σ が 200px 前後になり（切り出し帯 807×460 → σ=201.75）、真の畳み込みでは §5.1 の性能要件（1秒以内）を満たせない。
   このため本アプリでは**ボックスぼかし近似**を採用し、実装者が変わっても同一結果になるよう以下を確定仕様とする。

   | 項目 | 確定値 |
   |---|---|
   | 反復回数 | **3回**（本項の下限「3回反復以上」に合わせる） |
   | ボックス半径 | W3C SVG `feGaussianBlur` 仕様の式による。`d = floor(σ × 3 × √(2π) / 4 + 0.5)` とし、**d が奇数なら 3回とも d、d が偶数なら d, d, d+1** を用いる |
   | 端の扱い | 端の画素値を延長する（clamp）。帯の端が暗く落ち込むのを防ぐため |
   | 内部精度 | `Float32` で保持し、ぼかし処理の途中では丸めない。四捨五入は §3.1-4／§3.1-5 の最終画素値のみに適用する |
   | 実装手段 | 移動和（running sum）による O(画素数) の実装とし、σ が大きくても処理時間が増えないようにする |
3. 照明マップの全画素平均 L̄ を算出
4. 各画素を次式で補正する（RGB各チャンネルに**同一係数**を掛けることで色相を保持する）
   - `P'(x,y) = P(x,y) × L̄ / max(L(x,y), 1)`
   - 結果は 0〜255 にクリップし、四捨五入で整数化
5. **補正強度 k（0〜100%、既定100%）** を適用: `P''(x,y) = P(x,y) + (P'(x,y) − P(x,y)) × k/100`
   - k=0 は補正なし（現行v23と同一結果）であること

検算例: ある画素 P=(120, 100, 80)、その位置の L=180、L̄=144 のとき、係数 = 144/180 = 0.8、P' = (96, 80, 64)。k=50% なら P'' = (108, 90, 72)。

### 3.2 処理B: プロファイル押し出し（無地モード）

適用対象: フレーム種類「無地」選択時。処理Aの後に実行。

1. 切り出し帯（処理A適用済み）の長手方向を判定する（§2）
2. 断面方向の各位置 i について、長手方向に並ぶ全画素を対象に、RGB各チャンネル独立で**トリム平均**を取る
   - トリム平均: 対象画素をチャンネル値で昇順に並べ、下位10%・上位10%（それぞれ `floor(n×0.1)` 個）を除外した残りの算術平均。四捨五入で整数化
   - 目的: 撮影時の映り込み・ゴミ等の外れ値除去
3. 得られたプロファイル列（横帯なら 1×sh、縦帯なら sw×1）を、長手方向に必要長さぶん引き延ばしたものを辺テクスチャとする
   - 現行の `tileTopBot` / `tileSide`（転置）の仕組みに載せる。転置ロジックは現行v23のものを流用する
4. 無地モードでは「テクスチャスケール」スライダーは無効化（グレーアウト）する。プロファイル押し出しに繰り返し周期は存在しないため

期待効果: 照明ノイズ・継ぎ目が原理的にゼロになる（長手方向に一様なため）。

### 3.3 処理C: ミラータイリング（柄物モード）

適用対象: フレーム種類「柄物」選択時、かつ**ミラータイリングON時**（既定ON）。処理Aの後に実行。

1. 現行 `buildLongStrip()` のタイル配置において、**奇数番目のタイル（0起点で i=1,3,5…）を長手方向に反転**して描画する
   - 横ストリップ: 水平反転（`scale(-1,1)`）
   - 縦ストリップ: 垂直反転（`scale(1,-1)`）
2. 偶数番目タイルは現行どおり正像で描画する
3. これによりタイル境界で画素が鏡映一致し、継ぎ目の不連続が消える
4. **ミラータイリングOFF時は現行v23のタイル配置（全タイル正像）をそのまま用いる**（→ §4.1 の切替UI）

注記: 鏡映により模様の向きが1タイルおきに反転する。装飾フレームの模様は概ね対称であるため受容するが、方向性の強い模様（文字・矢印等）では不自然になり得る。このためON/OFF切替UIを設ける（§9 未確定事項 No.1 で確定済み）。

**後方互換との関係（2026-07-20追記）**: `buildLongStrip()` は繰り返し単位を `count = ceil(targetLen / step) + 1` で決めるため、テクスチャスケール100%でも必ず2枚以上のタイルを並べる。したがって i=1 のタイルが常に存在し、ミラーON時の描画結果は v23 と一致しない。**§7 T-2（後方互換）の判定はミラーOFF条件で行う**。

### 3.4 処理D: 4点パース補正（後続Step・本追記では仕様のみ確定）

1. 現行の2点矩形指定に代わり、フレーム辺の**4隅を順にクリック**（左上→右上→右下→左下）
2. 4点で囲まれた四辺形を射影変換（ホモグラフィ）で長方形にリサンプリングし、以後の処理A〜Cの入力とする
3. 出力長方形のサイズ: 幅 = 上辺・下辺の平均長、高さ = 左辺・右辺の平均長（いずれも四捨五入）
4. 2点指定モードも残し、UIで切替可能とする（メーカー画像では2点の方が速いため）

### 3.5 処理E: シミュレート結果のエクスポート

適用対象: シミュレーション実行後（プレビューCanvasに結果が描画されている状態）。

1. **エクスポート用Canvasをプレビューとは別に生成し、プレビューの2倍スケールで再レンダリング**する
   - `drawPreview()` の描画ロジックを共通関数化し、スケール係数を引数で受け取れるようにする（プレビュー=1、エクスポート=2）
   - Canvasを単純拡大コピーするのではなく、2倍解像度で描画し直すこと（ぼけ防止）
2. **設定情報キャプション帯**（既定ON、チェックボックスでOFF可）
   - エクスポート画像の下部に白地の帯を追加し、以下を黒文字で焼き込む
     - 1行目: 出力日時（`yyyy/MM/dd HH:mm`）＋「額装シミュレーション（イメージ）」
     - 2行目: 額縁種類（デッサン・水彩額／油彩額）、作品サイズ（幅×高さ mm）
     - 3行目: デッサン水彩の場合 → マット幅mm・マット色（HEX値）・面金（なし／色・幅mm）。油彩の場合 → ライナー・入子種類・幅mm
   - フォント: 端末標準サンセリフ、サイズはエクスポート画像幅の 2.2%（四捨五入）、行間1.5
   - 帯の高さ: テキスト3行が収まる高さ＋上下パディング各1行分
3. **免責文言の焼き込み（必須・OFF不可）**
   - キャプション帯の最下行に小さく「※本画像はシミュレーションによるイメージです。実物とは色味・質感が異なる場合があります。」を必ず含める
   - キャプション帯OFF時も、この免責文言のみ画像右下に半透明（不透明度60%）で焼き込む
4. **ファイル形式・ファイル名規則**
   - 形式: PNG（`canvas.toBlob('image/png')`）
   - ファイル名: `額装シミュレーション_{幅}x{高さ}mm_{yyyyMMdd-HHmmss}.png`（例: `額装シミュレーション_297x210mm_20260718-143025.png`）
5. **ダウンロード方式**
   - Blob URL＋`<a download>` によるダウンロードを基本とする
   - `download` 属性が機能しない環境向けフォールバック: 新規タブで画像を開き「長押しで保存してください」の案内を表示（iPad Safari対策）
6. 外部通信・サーバー保存は行わない（すべてクライアント内で完結）

- 単一HTMLファイル・追加費用ゼロ・外部通信なしを維持
- 処理A＋B（またはA＋C）の合計処理時間: 切り出し帯 1000×400px 程度で 1秒以内（一般的なPC/iPad）

- **Step 1**: 無地モード（処理B）＋種類切替UI
  - 完了条件: ①コード生成 ②本書との突合セルフレビュー報告 ③検収テスト §7 のうち T-1・T-2 クリア
- **Step 2**: 照明ムラ補正（処理A）＋ミラータイリング（処理C）＋補正UI
  - 完了条件: ①コード生成 ②突合セルフレビュー報告 ③検収テスト T-3〜T-6 クリア
- **Step 3**: 4点パース補正（処理D）
  - 完了条件: ①コード生成 ②突合セルフレビュー報告 ③検収テスト T-7 クリア

  - 完了条件: ①コード生成 ②突合セルフレビュー報告 ③検収テスト T-8〜T-10 クリア

## 7. 検収テストケース（実データ）

検収用画像は以下の2枚を正とする。
- 画像①: `test-images/frame1.jpg`（メーカー提供コーナーサンプル・柄物・均一照明）
- 画像②: `test-images/IMG_3569.jpg`（現場スマホ撮影・無地黒＋銀縁・照明勾配あり）

### 7.1 定量検証関数
`checkStripUniformity()` をブラウザconsoleから実行可能に実装する。
- 内容: 生成済み辺テクスチャ（上辺用ロングストリップ）について、長手方向の位置ごとの平均輝度 Y を算出し、その**最大値−最小値（ΔY）**を返す
- 8bit輝度スケール（0〜255）で評価する

### 7.2 テスト項目

| No | 条件 | 期待結果 |
|---|---|---|
| T-1 | 画像②・無地モード・補正ON・k=100 | ΔY ≤ 5。プレビュー上で明暗の縞・継ぎ目が視認できない |
| T-2 | 画像①・柄物モード・補正OFF・**ミラータイリングOFF**・スケール100% | v23の描画結果と画素一致（後方互換）。ミラーONではタイル配置が変わるため一致しない（§3.3 後方互換との関係を参照） |
| T-3 | 画像②・柄物モード・補正ON・k=100 | 補正OFF時と比べ ΔY が 1/3 以下に減少 |
| T-4 | 画像①・柄物モード・補正ON・k=100 | 模様の形状が破綻しない（彫刻パターンの山数・配置が元画像と目視一致） |
| T-5 | 補正強度 k=0 | 補正OFFと同一結果（画素一致） |
| T-6 | 柄物モード・ミラータイリングON | タイル境界位置で左右画素が鏡映一致していること（境界±1pxの輝度差 ≤ 2）。測定は検証ハーネス側で行い、製品コードに測定関数は追加しない（常設するのは §7.1 の `checkStripUniformity()` のみ） |
| T-7 | 画像②・4点指定（斜め撮影部を指定） | 矯正後の帯でフレーム辺が水平/垂直になり、T-1と同等の結果が得られる |
