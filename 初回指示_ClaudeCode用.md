# Claude Code 初回指示プロンプト

## 事前準備（手動・1回だけ）

1. このフォルダをローカルの開発用ディレクトリに配置（例: `C:\dev\gakuso-simulator`。半角英数字パス推奨）
2. GitHubで **privateリポジトリ `gakuso-simulator`** を作成（README等は追加しない・空で作成）
3. フォルダ内でターミナルを開き、Claude Code を起動

## 初回セッションで貼り付けるプロンプト

```
CLAUDE.md と 要件定義書_v24_画像前処理・エクスポート.md を読んでから作業してください。

まず以下を実施:
1. git init し、初回コミット「v23: 現行版をリポジトリ管理開始」を作成、
   リモート https://github.com/{アカウント名}/gakuso-simulator.git に push、
   タグ v23 を付与
2. その後、Step 1（無地モード: プロファイル押し出し）の実装計画を提示

計画には以下を含めること:
- 設計書§3.1〜3.2のどの仕様をどの関数に実装するか
- 既存コード（buildTex / buildLongStrip / drawPreview）への変更点と後方互換（T-2）の担保方法
- 検収テスト T-1・T-2 の確認方法

計画を提示したら、私の承認を待ってから実装を開始してください。
```

※ `{アカウント名}` は自分のGitHubアカウント名に置き換える。

## 2セッション目以降（会話が長くなったとき・翌日再開時）

```
CLAUDE.md と設計書を読み直してから続けてください。
現在の進捗は README.md の進捗表を参照。
```

## 各Step完了時の流れ（開発チェックリスト準拠）

1. 実装AIから突合セルフレビュー報告（設計書該当箇所→実装箇所の対応表）を受ける
2. 自分の目で要点確認 → 承認
3. 検収テスト該当項目（T-n）のクリアを確認
   - ブラウザconsoleで `checkStripUniformity()` を実行し ΔY を確認
4. README の進捗表・完了確認手順を更新させる
5. コミット・push を承認（メッセージ例: `Step1: 無地モード（プロファイル押し出し）実装`）
6. 実機（iPad Safari含む）確認後、タグ `v24-step1` を付与させる

## 補足

- Step 4（エクスポート）を先に使いたい場合は「Step 1 の次に Step 4 を前倒しで実施」と指示すればよい（CLAUDE.mdで許可済み）
- 仕様を変えたくなったら、コードより先に要件定義書の該当節を更新させること（設計書ファースト）
